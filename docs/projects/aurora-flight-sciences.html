Aurora Flight Sciences

Software Intern — Embedded Collision Avoidance for Autonomous UAVs

Internship
Summer 2016
Cambridge, MA

Overview

At Aurora Flight Sciences (now a Boeing company), I worked as a software intern on the embedded software team developing autonomous quadrotor UAVs. The primary project involved integrating an ultrasonic collision-avoidance sensor module into the vehicle's real-time control pipeline, requiring low-latency I/O handling and rigorous testing under flight-critical constraints.

Collision Avoidance Integration

The core project was the end-to-end integration of an ultrasonic collision-avoidance module for autonomous quadrotors. The sensor provided range measurements to nearby obstacles, and the challenge was processing this data fast enough for the flight controller to react in time:

  • C++ I/O pipeline optimization: Refactored the real-time I/O pipeline handling ultrasonic sensor data, reducing obstacle reaction latency from 150 ms to 30 ms — an 80% reduction — while maintaining jitter within ±2 ms bounds.
  • Real-time constraints: The pipeline operated under strict real-time requirements where missed deadlines could result in collision. The optimization focused on eliminating unnecessary data copies, reducing system call overhead, and streamlining the sensor-to-actuator data path.
  • Sensor integration: Handled the full integration stack from low-level serial communication with the ultrasonic module through data parsing, filtering, and feeding processed range data into the flight controller's obstacle avoidance logic.

Test & Verification

Beyond the integration work, I contributed to the team's verification process:

  • Test procedure authoring: Wrote formal test procedures and execution checklists for the collision-avoidance subsystem, establishing repeatable verification workflows.
  • Verification efficiency: The authored procedures directly reduced the system verification cycle time by 15%, enabling faster iteration between software changes and validated test results.
